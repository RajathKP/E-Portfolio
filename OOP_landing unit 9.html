Unit 9: Packaging and Testing

Unit Overview

This week’s unit focuses on bringing everything we’ve learned so far together by looking at how to properly package and test Python programs before sharing them with others. Packaging helps make a program easier to distribute and install, especially when it needs to be shared with stakeholders or end users. Alongside this, we explore the importance of code documentation using the PEP257 convention, which ensures that code is clearly explained and easy for others to understand or maintain.

Unit Reflection

Unit 9 focuses on the essential final stages of software development—packaging and testing Python programs. It also emphasizes the importance of clear documentation through the PEP257 convention, ensuring that programs are well-organised and understandable to others.

e-Portfolio Activities

This e-portfolio activities focus on understanding cyclomatic complexity and its relevance in object-oriented programming. The tasks explore how this metric helps measure the complexity of code by analysing the number of independent paths, while also encouraging reflection on alternative metrics that may better represent real-world program complexity. Additionally, the activities include practical coding exercises to apply these concepts and test program accuracy using the Python `assert` statement.
