Unit 3: Unified Modelling Language (UML)

Unit Overview

This unit introduces the Unified Modelling Language (UML), a standardised visual language used to design and represent software systems before implementation. UML provides a common framework for developers, analysts, and stakeholders to communicate system requirements, design concepts, and functional details effectively.

Unit Reflection

Unit 3 helps to understand how the Unified Modelling Language (UML) is used to visualise and design object-oriented systems and different types of UML diagrams, such as Use Case, Class, and State Machine diagrams, support various stages of the SDLC..

UML Models and the SDLC Stages

Different UML models are used at various stages of the Software Development Life Cycle (SDLC) to represent the system from different perspectives. During the requirements stage, Use Case Diagrams help capture user interactions and system functionality, while Activity Diagrams describe workflows. In the design stage, Class Diagrams define the system’s structure and relationships between objects, and Sequence Diagrams illustrate how objects interact over time. During implementation, Component and Deployment Diagrams represent the system’s physical and software architecture. Together, these models provide a complete visual understanding of the system before coding begins.
